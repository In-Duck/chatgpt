# MapleLand 자동화 프로그램

메이플랜드 게임을 위한 자동화 도구입니다.

## 주요 기능

- **창 모니터링**: 특정 창이 포그라운드에 있을 때만 자동화 실행
- **자동 줍기**: 설정한 간격으로 자동으로 키 입력
- **버프 자동화**: 최대 3개의 버프 키를 독립적으로 자동 입력
- **유저 탐지**: 특정 구역에서 빨간색 픽셀 감지 시 텔레그램 알림
- **핫키 지원**: 모든 기능을 핫키로 빠르게 제어
- **시스템 트레이**: 트레이로 최소화하여 백그라운드 실행
- **자동 업데이트**: GitHub Release를 통한 원클릭 업데이트

## 설치 방법

### 1. 필수 프로그램 설치
- Python 3.8 이상

### 2. 의존성 설치
```bash
pip install -r requirements.txt
```

### 3. 프로그램 실행
```bash
python main.py
```

## 사용 방법

### 초기 설정
1. **환경설정** 버튼 클릭
2. **기본 설정** 탭에서 모니터링할 창 선택
3. 줍기 키와 간격 설정
4. **버프 설정** 탭에서 버프 키와 간격 설정
5. **유저 탐색** 탭에서 텔레그램 설정 및 탐지 구역 설정
6. **핫키 설정** 탭에서 원하는 핫키 지정
7. **저장** 버튼 클릭

### 기능 사용
- **감지 시작**: 창 모니터링 시작 (설정한 창이 활성화될 때만 자동화 실행)
- **줍기 시작**: 자동 줍기 시작
- **버프1/2/3**: 각 버프 키 자동 입력 시작
- **유저탐색**: 특정 구역에서 유저 감지 시작
- **일괄 시작/중지**: 모든 기능 한 번에 제어

### 핫키 (기본값)
- `F9`: 줍기 토글
- `F10`: 모든 버프 토글
- `F11`: 창 감지 토글
- `F12`: 유저 탐색 토글

## 업데이트 방법

### 자동 업데이트 (권장)
1. 프로그램 시작 시 자동으로 업데이트 확인
2. 업데이트가 있으면 알림 표시
3. "예" 클릭 시 자동으로 다운로드 및 설치
4. 또는 **환경설정 > 업데이트** 탭에서 수동으로 확인 가능

### 수동 업데이트
1. [GitHub Releases](https://github.com/In-Duck/MapleLand/releases) 페이지 방문
2. 최신 버전의 `MapleLand-v*.*.*.zip` 다운로드
3. 압축 해제 후 기존 파일 덮어쓰기

## GitHub에 업로드하는 방법 (개발자용)

### 1. 저장소 초기화
```bash
cd C:\Users\rkddl\Desktop\MGX
git init
git remote add origin https://github.com/In-Duck/MapleLand.git
```

### 2. 파일 추가 및 커밋
```bash
git add .
git commit -m "Initial commit"
git branch -M main
git push -u origin main
```

### 3. 릴리즈 생성
1. GitHub 저장소 페이지에서 **Releases** 클릭
2. **Create a new release** 클릭
3. **Tag version**: `v1.0.0` 형식으로 입력
4. **Release title**: 버전 이름 입력 (예: `v1.0.0 - 초기 릴리즈`)
5. **Description**: 변경 사항 작성
6. **Attach binaries**: 프로그램 파일을 ZIP으로 압축하여 업로드
   - 파일명: `MapleLand-v1.0.0.zip`
   - 포함할 파일: 모든 `.py` 파일, `requirements.txt`, `README.md`, `version.json`
7. **Publish release** 클릭

### 4. 업데이트 릴리즈 생성 (버전 업데이트 시)
1. `version.json` 파일의 버전 번호 수정
2. 변경된 파일들을 커밋 및 푸시
3. 위의 3번 과정 반복 (새 버전 번호로)

## 문제 해결

### 업데이트가 안 될 때
- 인터넷 연결 확인
- GitHub 저장소가 공개(Public)로 설정되어 있는지 확인
- `version.json` 파일이 올바른지 확인

### 프로그램이 실행되지 않을 때
```bash
pip install -r requirements.txt --upgrade
```

### 핫키가 작동하지 않을 때
- 관리자 권한으로 실행
- 다른 프로그램과 핫키 충돌 확인
